# WorkMailReport 1차 / 2차 구현 계획서

## 프로젝트 개요

WorkMailReport는 로컬 PC 또는 사내 접근 가능한 환경에서 MSSQL 점검 쿼리를 실행하고, 결과를 HTML 메일로 정리해 지정된 수신자에게 발송하는 WinForms 기반 관리 프로그램이다.

초기 목표는 DB, 쿼리, 수신자, 메일 설정을 UI에서 관리하고, 실행 결과와 발송 이력을 확인할 수 있도록 구성하는 것이다.

## 개발 방향

- IDE: Visual Studio
- Language: C#
- Framework: .NET 8
- UI: WinForms
- Local Storage: SQLite
- MSSQL Access: Microsoft.Data.SqlClient
- Mail: MailKit
- Logging: SQLite 로그 테이블 또는 Serilog

초기에는 WinForms 단독 프로그램으로 개발하고, 향후 필요 시 실행부를 Windows Service로 분리한다.

## 1차 구현 범위

1차 구현 목표는 버튼으로 쿼리를 실행하고, 결과 메일을 발송하고, 로그를 확인할 수 있는 상태까지 만드는 것이다.

포함 기능:

- DB 접속 정보 관리
- MSSQL 연결 테스트
- 점검 쿼리 관리
- 쿼리 테스트 실행
- 쿼리 실행 결과 Grid 표시
- 메일 수신자 관리
- SMTP 메일 설정 관리
- 메일 즉시 발송
- 실행 로그 저장
- 쿼리 실패 로그 저장
- 메일 발송 로그 저장

1차 제외 기능:

- 정교한 예약 스케줄러
- Windows Service 분리
- 권한 관리
- 대시보드 차트
- 메일 템플릿 편집기
- 대량 재발송
- 사용자별 로그인

2차 구현 목표는 실제 운영에 가깝게 사용할 수 있도록 예약 실행, 메일 미리보기, 재발송, 실패 로그 처리 기능을 추가하는 것이다.

포함 기능:

- 실행 시간 설정
- 예약 실행 자동 수행
- 메일 미리보기
- 실행 결과 기반 메일 제목 자동 생성
- 메일 재발송
- 실패 로그 처리 완료 기능
- 실패 건 재실행
- 실행 이력 상세 조회
- 메일 본문 로그 조회

선택 기능:

- Windows 시작 시 자동 실행
- Tray Icon 최소화
- 프로그램 중복 실행 방지
- 설정 백업/복원
- 쿼리 결과 Excel Export
- Windows Service 분리

## 로컬 저장 테이블 초안

SQLite에는 아래 정보를 저장한다.

- Databases: DB 접속 대상 정보
- QueryChecks: 점검 쿼리 정보
- MailRecipients: 메일 수신자 정보
- MailSettings: 메일 발송 설정
- ScheduleSettings: 예약 실행 설정
- ExecutionRuns: 실행 단위 로그
- QueryExecutionLogs: 쿼리별 실행 로그
- MailSendLogs: 메일 발송 로그
- ErrorLogs: 실패 및 오류 로그

## 주요 서비스 설계

### QueryRunner

- MSSQL 연결 생성
- SQL 실행
- 결과 반환
- 제한 시간 처리
- 오류 발생 시 로그 저장

### CheckResultEvaluator

- 쿼리 실행 결과 판단
- SUCCESS, WARNING, FAILED 상태 결정
- 판단 방식 기반 처리

### ReportBuilder

- 쿼리 결과를 HTML 메일 본문으로 변환
- 요약 영역 생성
- 상세 결과 테이블 생성
- 오류 목록 생성

### MailSender

- SMTP 설정 조회
- 수신자 조회
- HTML 메일 발송
- 발송 로그 저장
- 실패 시 오류 로그 저장

### ExecutionService

- 실행 흐름 전체 관리
- 실행 로그 생성 및 업데이트
- 쿼리 실행
- 결과 판단
- 메일 발송 호출
- 실행 결과 요약 저장

### ScheduleService

2차 구현에서 추가한다.

- 예약 설정 조회
- 현재 시간이 실행 대상인지 판단
- 중복 실행 방지
- 예약 실행 호출
